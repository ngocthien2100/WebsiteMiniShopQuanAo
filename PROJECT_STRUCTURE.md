# Project structure

This project is a Vite + React frontend for the MiniStyle clothing shop demo.

## Active source code

```text
src/
  main.tsx                 App bootstrap.
  app/
    App.tsx                Main shop shell, routing state, cart.
    App.css                Main application styling and animation.
  features/
    admin/                 Admin dashboard: analytics, product CRUD, users.
    auth/                  Login and register pages.
    customer/              Customer profile and order history.
    staff/                 Staff product management screen.
  shared/
    data/                  Product catalog, mock users, mock orders, localStorage DB helpers.
    services/              Data repositories for Supabase with localStorage fallback.
    supabase/              Frontend Supabase client configuration.
  styles/                  Global CSS, Tailwind entry, theme and Georgia/Verdana font config.
```

## Documentation

```text
docs/
  chatbot-test-cases.md    Chatbot demo and test prompts.
  database-supabase-guide.md Supabase database, Vercel env, and workflow data access guide.
  prompt-library.md        Prompt examples for chatbot/product advice.
```

## Database

```text
supabase/
  schema.sql               Tables, Row Level Security policies, and seed data.
```

## Generated archive

```text
archive/generated/
  figma-imports/           Old Figma-exported page snapshots and image assets.
  figma-components/        Old helper generated by Figma export.
  shadcn-ui/               Unused generated shadcn/ui components kept for reference.
  template-bootstrap/      Unused original generated template files.
  old-package-manager/     Old pnpm files; npm is the active package manager.
```

Files in `archive/generated` are not imported by the running application. They are kept outside `src` so reviewers can focus on the actual implementation first.

## Root config

```text
index.html                 Vite HTML shell.
vite.config.ts             Vite, React, Tailwind, and path alias config.
vercel.json                Vercel SPA rewrite config.
package.json               Scripts and dependencies.
```
